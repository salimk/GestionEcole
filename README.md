# Projet pédagogique Spring Boot – Système académique
**4ème année - Filière IDSI -Ecole EMSI - 2026**

## Présentation générale
Ce dépôt GitHub regroupe un projet pédagogique réalisé dans le cadre de l’enseignement du développement d’applications d’entreprise. Il sert de support global pour accompagner les étudiants dans la découverte progressive des concepts, outils et bonnes pratiques liés à la conception et à la réalisation d’une application basée sur l’écosystème Spring Boot.
Le projet s’appuie sur un cas académique cohérent permettant de mettre en œuvre une architecture applicative claire, modulaire et évolutive. Il a été pensé comme un fil conducteur pédagogique, destiné à illustrer progressivement les différentes couches d’une application moderne, depuis la structuration du projet jusqu’à l’implémentation des fonctionnalités métier.

## Objectifs pédagogiques
Ce projet a pour finalité de permettre aux étudiants de :
- comprendre l’organisation générale d’une application d’entreprise ;
- adopter une approche de développement structurée par couches ;
- manipuler les principaux composants de l’écosystème Spring ;
- relier les notions théoriques du module à une mise en pratique concrète ;
- construire progressivement une application complète dans un cadre pédagogique guidé.

## Organisation pédagogique du dépôt
Ce dépôt constitue le support principal du projet global. Les différentes étapes de réalisation et d’enrichissement du projet sont destinées à être structurées sous forme de versions successives, afin de refléter la progression pédagogique du module et de faciliter le suivi des différentes phases d’apprentissage.

## Domaine fonctionnel
Le projet prend pour contexte un environnement académique, choisi pour sa simplicité, sa cohérence et sa proximité avec les besoins d’apprentissage des étudiants. Ce cadre permet de construire un système représentatif d’une application métier tout en conservant une complexité adaptée à un usage pédagogique.

## Note importante
Ce projet possède avant tout une **finalité pédagogique**. Il a été conçu comme un **complément de la formation**, afin d’aider les étudiants à relier les notions théoriques étudiées en cours à une implémentation concrète et progressive.
Le code proposé dans ce dépôt est volontairement **documenté** et structuré de manière à favoriser la compréhension, la lecture et l’apprentissage. Il ne doit donc pas être interprété comme la reproduction d’un produit industriel finalisé, ni comme un projet professionnel intégrant l’ensemble des contraintes techniques, organisationnelles, sécuritaires, de performance, de test, de déploiement ou de maintenabilité attendues dans un environnement réel de production.
L’objectif principal de ce dépôt est de servir de **support d’apprentissage**, et non de constituer une référence complète de réalisation professionnelle.

## Public concerné
Ce projet est destiné aux étudiants de **4ème année – filière IDSI** à l’**EMSI**, dans le cadre de leur formation en développement logiciel et en conception d’applications d’entreprise.
